Project Summary: Personal Doctor System

Technical Skills: Core JAVA, MySQL

INTRODUCTION TO THE PROJECT

  • Objective: The Personal Doctor project aids users in diagnosing diseases based on symptoms, maintaining detailed disease and medicine databases.
  • Functionality: Allows quick disease diagnosis, stores disease and medicine information, and provides urgent assistance by suggesting medicines.
  • Evolution: Originally focused on disease information, it now includes medicine details, compositions, and diet suggestions.

SCOPE

  • Future Expansion: Options left open for potential enhancements, including web-based and mobile applications, additional user authorities, email features, sound integration, and client-server architecture.
  • Code Reusability: Emphasizes that a web-based application can reuse most of the existing code.

PROPOSED SYSTEM

  • Stores information for diseases, symptoms, medicines, diet suggestions, and related details.
  • Enhances maintenance, flexibility, and centralized data storage to reduce redundancy and enhance consistency.

SOFTWARE REQUIREMENTS

  • Skill Set: Utilizes Java for platform independence and compact size.
  • Purpose: Software Requirements Specification (SRS) ensures alignment with customer needs and guides developers, maintenance engineers, and user document writers.

SYSTEM FEATURES

  • Diagnostic Assistance: Enables disease diagnosis based on symptoms.
  • Information Storage: Manages databases for diseases, symptoms, medicines, and related details.
  • User Roles: Admin and user modes with varied permissions.
  • Hardware and Software Requirements: Specifies processor, RAM, monitor, keyboard, mouse, hard disk, front-end (Core Java), back-end (MySQL), and operating system requirements.

SYSTEM DESIGN AND MODELING

  • Module Description: Ensures only authorized users modify records, with modules for login, disease info, symptoms, medicines, and more.
  • Function-Oriented Design: Presents 0-Level DFD and data dictionary for login information, disease details, symptoms, medicines, and other related entities.

CONCLUSION

  • System Evolution: From a disease-focused system to a comprehensive Personal Doctor tool.
  • Future Potential: Can serve as a crucial medical assistance tool if extended, helping individuals in need.
  • Design Highlights: Emphasizes user-friendliness, robustness, and controlled access for administrators and users.
